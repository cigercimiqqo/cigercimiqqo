03
Firestore Güvenlik Kuralları
Aşağıdaki kodu kopyalayıp Firebase'e yapıştır
  • → Firebase → Sol menü → Firestore Database → Rules sekmesi
  • → Mevcut tüm içeriği sil, aşağıdaki kodu yapıştır
  • → Publish butonuna bas
Firestore Rules 
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // Ayarlar, kategoriler, ürünler, blog — herkese okunabilir
    match /settings/{doc}    { allow read: if true; allow write: if request.auth != null; }
    match /categories/{doc}  { allow read: if true; allow write: if request.auth != null; }
    match /products/{doc}    { allow read: if true; allow write: if request.auth != null; }
    match /blog_posts/{doc}  { allow read: if true; allow write: if request.auth != null; }
    match /pages/{doc}       { allow read: if true; allow write: if request.auth != null; }

    // Siparişler — herkes yazabilir, sadece admin okuyabilir
    match /orders/{doc} {
      allow create: if true;
      allow read, update, delete: if request.auth != null;
    }

    // Yorumlar — herkes ekleyebilir, admin yönetebilir
    match /reviews/{doc} {
      allow create: if true;
      allow read: if true;
      allow update, delete: if request.auth != null;
    }

    // Ziyaretçiler, kuponlar — sadece admin
    match /visitors/{doc} { allow read, write: if request.auth != null; }
    match /coupons/{doc}  { allow read: if true; allow write: if request.auth != null; }
  }
}

08
Terminalden Build Al ve Cloudflare'e Deploy Et
GitHub yok — 3 komut, direkt Cloudflare'e gönder
⚡ En temiz yöntem bu. Terminalde 3 komut çalıştırırsın, proje direkt Cloudflare Pages'e yüklenir. GitHub'a, Desktop uygulamasına gerek yok.
  • → Terminal aç → proje klasörüne git:
Terminal 
cd /Users/furkangunduz/CursorProjects/MiqqoSite && npm install
  • → Bu komut projeyi Cloudflare Pages formatına dönüştürür:
Terminal 
npx @cloudflare/next-on-pages@1
  • → Bittikten sonra .vercel/output/static klasörü oluşur — burası deploy edilecek
  • → miqqo-musteri-adi kısmını müşterinin kısa adıyla değiştir:
Terminal — ilk deploy 
npx wrangler pages deploy .vercel/output/static --project-name miqqo-musteri-adi
  • → İlk çalıştırmada Cloudflare hesabıyla giriş isteyecek — tarayıcı açılır, onayla
  • → Proje yoksa otomatik oluşturur, varsa günceller
  • → Bittikten sonra sana bir URL verir: https://miqqo-musteri-adi.pages.dev
🔄 Sonraki güncellemelerde sadece şu 2 komutu çalıştır:
Terminal — güncelleme deploy 
npx @cloudflare/next-on-pages@1 && npx wrangler pages deploy .vercel/output/static --project-name miqqo-musteri-adi
09
Cloudflare Pages'te nodejs_compat Flag Ekle
Firebase SDK'nın çalışması için zorunlu — tarayıcıdan yapılıyor
  • → dash.cloudflare.com → Workers & Pages → az önce oluşan projeyi seç
  • → Settings → Functions sekmesi
  • → Compatibility flags bölümü → Add compatibility flag
  • → nodejs_compat yaz → Save
  • → Deployments → Retry deployment
✅ Bu flag Firebase SDK'nın Cloudflare edge ortamında sorunsuz çalışmasını sağlar. Bir kere eklersin, her deploy'da otomatik aktif olur.
  • → Aynı sayfada Compatibility date bölümü → en az 2024-09-23 olsun
  • → Değilse güncelle → Save

11
Müşteri Domainini Bağla
Cloudflare Pages → Custom Domains
  • → Cloudflare Pages → Projen → Custom domains → Set up a custom domain
  • → Domain adını yaz → Continue → Otomatik ayarlanır → Bitti! SSL de otomatik gelir.
  • → Cloudflare Pages → Projen → Custom domains → Set up a custom domain
  • → Domain adını yaz → Cloudflare sana hedef adresi gösterir, örn: miqqo-musteri.pages.dev
  • → Domain sağlayıcına gir → DNS yönetimi → CNAME kaydı ekle:
DNS CNAME Kaydı 
Type:   CNAME
Name:   @  (veya www)
Value:  miqqo-musteri-adi.pages.dev
TTL:    Auto
  • → 1-24 saat içinde aktif olur
  • → Kontrol: dnschecker.org → domain gir → CNAME'in yayıldığını görünce hazır
🔒 Cloudflare ücretsiz SSL sertifikası otomatik sağlar. Müşteri sitesi https:// ile açılır.

Sık Karşılaşılan Sorunlar
❌ Admin login çalışmıyor ▾
Sebep: Firestore Rules yanlış yapıştırılmış veya Publish edilmemiş.

Çözüm: Firebase Console → Firestore Database → Rules sekmesi → Adım 3'teki kodu tekrar yapıştır → Publish butonuna bas.
❌ Görseller yüklenmiyor ▾
Sebep: Cloudinary preset "Signed" modunda ayarlı.

Çözüm: Cloudinary → Settings (⚙️) → Upload sekmesi → Upload preset'in üstüne tıkla → Signing mode: Unsigned yap → Save.
❌ Site açılıyor ama veri gelmiyor / boş ▾
Sebep: Cloudflare'deki env değişkenleri eksik veya yanlış girilmiş.

Çözüm: Cloudflare Pages → Projen → Settings → Environment variables → tüm değerleri kontrol et → Save → Deployments → Retry deployment.
❌ Domain bağlanmıyor / sitenin eski hali geliyor ▾
Sebep: DNS yayılımı henüz tamamlanmamış (24 saate kadar sürebilir).

Çözüm: dnschecker.org adresine git → domain adını yaz → CNAME kaydının yayıldığını gör → bekle. Cloudflare Pages → Custom domains → yeşil ✓ bekliyorsun.
❌ npm run build komutu hata veriyor ▾
Sebep: Node modülleri eksik veya bozuk.

Çözüm: Terminalde şu komutu çalıştır, ardından tekrar build et: 
npm install
