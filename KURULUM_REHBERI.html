Firebase Projesi Oluştur
Firestore, Realtime Database, Authentication ve Web App yapılandırmasını tamamlayın.
1a — Proje Oluşturma
  1. Firebase Console'a gidin ve Proje Ekle'ye tıklayın.
  2. Proje adını girin (ör: miqqo-musteri-adi).
  3. Google Analytics'i isteğe bağlı olarak etkinleştirin, ardından Proje Oluştur'a tıklayın.
1b — Firestore Database
  1. Sol menüden Firestore Database → Veritabanı oluştur'a tıklayın.
  2. Konum olarak europe-west1 (Belçika) seçin.
  3. Test modunda başlat'ı seçip oluşturun.
⚠️ Production'a geçmeden önce güvenlik kurallarını mutlaka güncelleyin.
1c — Realtime Database
  1. Sol menüden Realtime Database → Veritabanı oluştur'a tıklayın.
  2. Konum olarak europe-west1 seçin.
  3. Test modunda başlatın. URL'yi not alın (ör: https://proje-adi-default-rtdb.europe-west1.firebasedatabase.app).
1d — Authentication
  1. Sol menüden Authentication → Başlayın'a tıklayın.
  2. E-posta/Şifre sağlayıcısını etkinleştirin.
  3. Users sekmesinden ilk admin kullanıcıyı oluşturun.
1e — Web App Oluşturma
  1. Proje Ayarları → Uygulamalarınız bölümüne gidin.
  2. Web simgesine (</>) tıklayıp uygulama adı girin.
  3. Firebase config değerlerini kopyalayın — bunları .env.local dosyasında kullanacaksınız:
apiKey: "AIza..."
authDomain: "proje-adi.firebaseapp.com"
projectId: "proje-adi"
storageBucket: "proje-adi.appspot.com"
messagingSenderId: "123..."
appId: "1:123..."
databaseURL: "https://proje-adi-default-rtdb.europe-west1.firebasedatabase.app"

Proje Dosyalarını Hazırla
Bağımlılıkları yükleyin ve ortam değişkenlerini ayarlayın.

Bağımlılıkları Yükleme

Proje klasöründe terminali açın ve çalıştırın:

npm install

Ortam Değişkenleri (.env.local)

Proje kök dizinine .env.local dosyası oluşturun ve Firebase config değerlerini yapıştırın:

NEXT_PUBLIC_FIREBASE_API_KEY=AIzaSy...
NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=proje-adi.firebaseapp.com
NEXT_PUBLIC_FIREBASE_PROJECT_ID=proje-adi
NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET=proje-adi.appspot.com
NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=123456789
NEXT_PUBLIC_FIREBASE_APP_ID=1:123456789:web:abc123
NEXT_PUBLIC_FIREBASE_DATABASE_URL=https://proje-adi-default-rtdb.europe-west1.firebasedatabase.app

NEXT_PUBLIC_RESTAURANT_ID=ana-restoran

NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME=
NEXT_PUBLIC_CLOUDINARY_UPLOAD_PRESET=
NEXT_PUBLIC_IMGBB_API_KEY=
⚠️ .env.local dosyası .gitignore'da olmalı — asla commit etmeyin!

Görsel Yükleme Ayarları
Cloudinary veya ImgBB üzerinden görsel barındırma servisini yapılandırın.
  1. cloudinary.com'da ücretsiz hesap oluşturun.
  2. Dashboard'dan Cloud Name'i kopyalayın.
  3. Settings → Upload bölümünde Upload Preset oluşturun:
    • Signing Mode: Unsigned
    • Folder: miqqo
  4. .env.local dosyasına ekleyin:
NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME=your-cloud-name
NEXT_PUBLIC_CLOUDINARY_UPLOAD_PRESET=your-preset-name
  1. imgbb.com'da hesap oluşturun.
  2. api.imgbb.com'dan API anahtarı alın.
  3. .env.local dosyasına ekleyin:
NEXT_PUBLIC_IMGBB_API_KEY=your-api-key

GitHub Repository Oluştur
Projeyi GitHub'a yükleyin.

Yeni Repository

  1. github.com/new adresine gidin.
  2. Repository adı: miqqo-musteri-adi
  3. Private olarak oluşturun (isteğe bağlı).
  4. .gitignore ve README eklemeyin — proje zaten içeriyor.

Projeyi Push Edin

git init
git add .
git commit -m "🚀 initial commit"
git branch -M main
git remote add origin https://github.com/KULLANICI/REPO-ADI.git
git push -u origin main
ℹ️ KULLANICI ve REPO-ADI kısımlarını kendi bilgilerinizle değiştirin.
🔑 Farklı Hesaba Push — Token ile Yetkilendirme

Eğer müşterinin GitHub hesabına kendi bilgisayarından push yapıyorsan ve "Permission denied" veya "403" hatası alıyorsan, müşteri hesabının Personal Access Token (PAT)'ını kullanman gerekir:

1. Token Oluşturma

  1. Müşterinin GitHub hesabıyla giriş yap
  2. Sağ üst profil resmi → Settings
  3. Sol menüde en alta in → Developer settings
  4. Personal access tokens → Tokens (classic)
  5. Generate new token (classic)'e tıkla
  6. Not: miqqo-deploy
  7. Expiration: No expiration (veya istediğin süre)
  8. Scopes: repo kutusunu işaretle (tüm alt kutular otomatik seçilir)
  9. Generate token → çıkan token'ı hemen kopyala!
⚠️ Token sadece bir kere gösterilir! Kaybedersen yenisini oluşturman gerekir. Token ghp_ ile başlar.

2. Token ile Push

Remote URL'e token'ı ekleyerek push yap:

git remote set-url origin https://MUSTERI_KULLANICI:TOKEN@github.com/MUSTERI_KULLANICI/REPO-ADI.git
git push -u origin main
ℹ️ Örnek: Müşteri hesabı cigercimiqqo, token ghp_abc123... ise:

git remote set-url origin https://cigercimiqqo:ghp_abc123@github.com/cigercimiqqo/cigercimiqqo.git

3. Sonraki Push'lar

Token bir kere URL'e eklendikten sonra sonraki push'lar otomatik çalışır:

git add .
git commit -m "🔧 güncelleme"
git push

GitHub Pages Deploy
GitHub Actions ile otomatik deploy veya manuel gh-pages ile yayınlayın.

Adım A — Workflow Dosyası

Aşağıdaki dosyayı oluşturun: .github/workflows/deploy.yml

name: Deploy to GitHub Pages

on:
  push:
    branches: [ main ]

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: "pages"
  cancel-in-progress: false

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Node
        uses: actions/setup-node@v4
        with:
          node-version: "20"
          cache: "npm"

      - name: Install dependencies
        run: npm ci

      - name: Build
        run: npm run build
        env:
          NEXT_PUBLIC_FIREBASE_API_KEY: ${{ secrets.NEXT_PUBLIC_FIREBASE_API_KEY }}
          NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN: ${{ secrets.NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN }}
          NEXT_PUBLIC_FIREBASE_PROJECT_ID: ${{ secrets.NEXT_PUBLIC_FIREBASE_PROJECT_ID }}
          NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET: ${{ secrets.NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET }}
          NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID: ${{ secrets.NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID }}
          NEXT_PUBLIC_FIREBASE_APP_ID: ${{ secrets.NEXT_PUBLIC_FIREBASE_APP_ID }}
          NEXT_PUBLIC_FIREBASE_DATABASE_URL: ${{ secrets.NEXT_PUBLIC_FIREBASE_DATABASE_URL }}
          NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME: ${{ secrets.NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME }}
          NEXT_PUBLIC_CLOUDINARY_UPLOAD_PRESET: ${{ secrets.NEXT_PUBLIC_CLOUDINARY_UPLOAD_PRESET }}
          NEXT_PUBLIC_IMGBB_API_KEY: ${{ secrets.NEXT_PUBLIC_IMGBB_API_KEY }}

      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: ./out

  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    needs: build
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4

Adım B — next.config Ayarı

next.config.js (veya next.config.ts) dosyasında output: 'export' olduğundan emin olun:

/** @type {import('next').NextConfig} */
const nextConfig = {
  output: 'export',
  images: {
    unoptimized: true,
  },
};

module.exports = nextConfig;

Adım C — GitHub Secrets

Repository → Settings → Secrets and variables → Actions bölümüne gidin.

New repository secret ile her bir ortam değişkenini ekleyin:

NEXT_PUBLIC_FIREBASE_API_KEY
NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN
NEXT_PUBLIC_FIREBASE_PROJECT_ID
NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET
NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID
NEXT_PUBLIC_FIREBASE_APP_ID
NEXT_PUBLIC_FIREBASE_DATABASE_URL
NEXT_PUBLIC_RESTAURANT_ID
NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME
NEXT_PUBLIC_CLOUDINARY_UPLOAD_PRESET
NEXT_PUBLIC_IMGBB_API_KEY
⚠️ Secret adları birebir aynı olmalı — büyük/küçük harf duyarlıdır.

Adım D — GitHub Pages Kaynağını Ayarlayın

  1. Repository → Settings → Pages'e gidin.
  2. Source olarak GitHub Actions'ı seçin.
  3. Değişiklikleri commit & push edin — deploy otomatik başlayacak.
✅ Deploy tamamlandığında siteniz https://KULLANICI.github.io/REPO-ADI/ adresinde yayınlanacak.

Manuel Deploy (Alternatif)

Bu yöntemde gh-pages paketi ile build çıktısını doğrudan deploy edersiniz.

npm install --save-dev gh-pages

package.json'a deploy scripti ekleyin:

"scripts": {
  "predeploy": "npm run build",
  "deploy": "gh-pages -d out"
}

Deploy komutu:

npm run deploy

GitHub → Settings → Pages → Source: Deploy from a branch → Branch: gh-pages / root seçin.

ℹ️ Bu yöntemde her değişiklikte npm run deploy komutunu manuel çalıştırmanız gerekir.

Custom Domain Bağlama
Kendi alan adınızı GitHub Pages'e bağlayın (isteğe bağlı).

GitHub Pages Domain Ayarı

  1. Repository → Settings → Pages → Custom domain alanına domain girin (ör: menu.restoranadi.com).
  2. Enforce HTTPS'i aktifleştirin.

DNS Ayarları

Domain sağlayıcınızın DNS panelinde aşağıdaki kayıtları ekleyin:

Alt Alan Adı İçin (ör: menu.restoranadi.com)
Tür: CNAME
İsim: menu
Değer: KULLANICI.github.io
Kök Domain İçin (ör: restoranadi.com)
Tür: A
İsim: @
Değer: 185.199.108.153
       185.199.109.153
       185.199.110.153
       185.199.111.153
ℹ️ DNS değişiklikleri 24-48 saat sürebilir, genellikle birkaç dakika içinde aktif olur.

CNAME Dosyası

Proje kök dizinine public/CNAME dosyası oluşturun:

menu.restoranadi.com

Bu dosya deploy sırasında custom domain'in korunmasını sağlar.

Firebase Auth Domain Güncelle
Custom domain kullanıyorsanız Firebase'e bu domaini yetkilendirin.

Yetkilendirilmiş Domainler

  1. Firebase Console → Projeniz → Authentication → Settings
  2. Authorized domains bölümüne gidin.
  3. Add domain ile aşağıdaki domainleri ekleyin:
KULLANICI.github.io
menu.restoranadi.com
ℹ️ Hem GitHub Pages varsayılan domainini hem de (varsa) custom domaini ekleyin.

Sorun Giderme
Sık karşılaşılan sorunlar ve çözümleri.
Build hatası: "next export" komutu çalışmıyor

Next.js 14+ sürümlerinde next export komutu kaldırıldı. Yerine next.config.js'de output: 'export' kullanın. npm run build komutu otomatik olarak out/ klasörüne static export oluşturur.

GitHub Actions workflow başarısız oluyor

Kontrol edin:

  • Tüm secrets doğru eklenmiş mi? (Settings → Secrets)
  • Repository → Settings → Pages → Source "GitHub Actions" olarak seçili mi?
  • Repository → Settings → Actions → General → Workflow permissions: "Read and write" mi?
  • deploy.yml dosyasında YAML syntax hatası var mı?
Site açılıyor ama beyaz/boş sayfa gösteriyor

Muhtemel sebepler:

  • basePath eksik olabilir. Repository adı ile alt dizinde yayınlanıyorsa next.config.js'e ekleyin: basePath: '/REPO-ADI'
  • Custom domain kullanıyorsanız basePath gerekmez.
  • Tarayıcı konsol hatalarını kontrol edin (F12 → Console).
Firebase "auth/unauthorized-domain" hatası

Firebase Console → Authentication → Settings → Authorized domains listesine sitenizin domainini ekleyin. Hem kullanici.github.io hem de custom domain eklenmeli.

Sayfalar arası geçişte 404 hatası

GitHub Pages SPA routing desteklemez. Çözümler:

  • out/ klasörüne bir 404.html dosyası ekleyin (Next.js static export ile otomatik oluşturulabilir).
  • Veya next.config.js'e trailingSlash: true ekleyin — her sayfa kendi klasöründe index.html olarak oluşturulur.
Custom domain'de HTTPS çalışmıyor

DNS kayıtlarının yayılması biraz zaman alabilir. 30 dakika bekleyin, ardından GitHub → Settings → Pages'de "Enforce HTTPS" kutusunu tekrar işaretleyin. DNS doğrulaması başarılı olunca GitHub otomatik SSL sertifikası oluşturur.

Görseller yüklenmiyor (Cloudinary/ImgBB)

Kontrol edin:

  • API anahtarları .env.local ve GitHub Secrets'ta doğru mu?
  • Cloudinary'de upload preset Unsigned modunda mı?
  • ImgBB API anahtarı aktif mi?
  • Tarayıcı Network sekmesinden hata detayını kontrol edin.
Deploy sonrası değişiklikler görünmüyor

Çözümler:

  • Tarayıcı cache'ini temizleyin: Ctrl+Shift+R (Windows) veya Cmd+Shift+R (Mac).
  • GitHub Actions sekmesinden son workflow'un başarılı tamamlandığını doğrulayın.
  • GitHub Pages'in CDN cache'i birkaç dakika gecikebilir — 5 dakika bekleyin.
